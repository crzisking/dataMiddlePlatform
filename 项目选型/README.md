# 制造业数据中台 — 项目选型总览

> 从 0 搭建一个前后端系统，整合现有 OA / ERP / MES 等系统，提供两大能力：
> 1. **RAG**：基于企业非结构化文档的知识问答
> 2. **TextToSQL**：自然语言查询业务数据库，并可同时检索 RAG，回答混合型问题
>
> 前端有 Vue3 仓库与 TS+Vue 桌面端，**目前仅有一个入口页，管理页 / 上传页尚未开发**，待后端完成后再做。
> 因此采用**后端先行**：API 契约由后端从 0 定义，前端后续对接。本仓库聚焦后端与 AI 服务。

## 已确定的关键约束（全局）

| 约束 | 决策 |
|---|---|
| **LLM 部署** | 走云厂商 API：**通义千问 + DeepSeek**（OpenAI 兼容接口），**用户在对话中自选模型** |
| **Embedding** | **通义云 `text-embedding-v3`**（纯 API，不本地部署） |
| **后端语言** | **全部使用 Python** |
| **数据分布** | **业务数据全在 SQL Server**；**RAG 向量 + 中台元数据在 PostgreSQL**；二者不跨库 |
| **上传原件存储** | 现成 **MinIO** 对象存储 |
| **现有系统** | OA、ERP、MES 等 |
| **认证/权限** | 一期**不分用户、不做 API 鉴权**；中台为纯服务，用户身份由调用方记录 |
| **规模** | 用户 500~1000，并发 200~300 |
| **TextToSQL 能力** | 不仅查业务数据库，也可检索 RAG，支持混合型问题 |
| **部署** | 一期 Windows 服务器，后期迁 Docker |

## 整体架构分层

> 说明：管理页 / 上传页尚未开发（后端先行）；一期不做认证 / 鉴权，限流放到上线前硬化阶段（见 [工作清单.md](工作清单.md) P8）。

```
Web 入口页 (Vue3)  +  桌面端 (TS+Vue, 本地 SQLite)
            │ HTTP / SSE
┌───────────▼─────────────────────────────┐
│   API 网关层  (FastAPI)                   │
│   SSE 流式输出 / 模型路由(用户选模型)        │
│   (一期无认证鉴权；限流见 P8)               │
└───────────┬─────────────────────────────┘
            │
   ┌────────────┴────────────┐
   │  多轮对话 Agent (LangGraph) │ ← 模型自主决定调用哪个工具，可多步循环
   │  · 对话记忆(按会话ID)        │
   └────────┬─────────┬───────┘
       调用工具/skill │         │ 调用工具/skill
 ┌─────▼───┐ ┌──▼──────┐
 │TextToSQL│ │  RAG     │   ← 作为 Agent 可调用的工具
 │ 工具     │ │ 工具     │     详见各自文档
 └────┬────┘ └────┬─────┘
      │           │
 ┌────▼────────┐ ┌▼──────────────────────┐
 │ SQL Server   │ │ PostgreSQL            │
 │ 业务数据(只读 │ │  · PGVector 向量库     │
 │ 视图/账号)    │ │  · 对话/文档/配置元数据 │
 └─────────────┘ └───────────────────────┘
                  ┌───────────────────────┐
                  │ MinIO  上传原件对象存储 │
                  └───────────────────────┘
```
